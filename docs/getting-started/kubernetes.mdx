# Quickstart: Kubernetes Operator

This quickstart will use the [ngrok Kubernetes Operator](https://github.com/ngrok/kubernetes-ingress-controller) to put an application running on a Kubernetes Cluster on the internet and add some basic policies.

## What is the ngrok Kubernetes Operator?

The ngrok Kubernetes Operator is the best way to use ngrok if your applications run as Services in a Kubernetes cluster. Some of the benefits of the operator are:

- The Operator can be easily installed on a cluster via Helm and is compatible with the most recent versions of Kubernetes.
- The Operator supports both [Ingress CRDs](https://kubernetes.io/docs/concepts/services-networking/ingress/) as well as the new [Gateway API](https://gateway-api.sigs.k8s.io/) using the same installation.
- When deployed to a cluster, the Operator allows developers to self-service public endpoints to their apps and services using a shared ngrok account.
- The Operator manages the creation of ngrok resources, leaving developers to focus on traffic and routing patterns.
- The Operator supports the configuration of resources like custom domains, modules, and policies.
- The Operator supports all the cloud benefits of ngrok, including our global points of presense, GSLB, and traffic inspector.
